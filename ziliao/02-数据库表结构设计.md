# MP3分组播放系统 数据库表结构设计
**版本**：V1.0  
**更新时间**：2026-05-15  
**数据库**：MySQL 8.x  
**字符集**：utf8mb4  
**排序规则**：utf8mb4_unicode_ci

## 一、表关系图
```
User(用户) ←→ Group(分组)
     ↓
UserListened(用户已听记录) ←→ Audio(音频文件) ←→ Group(分组)
```

## 二、详细表结构

### 1. auth_group（Django自带分组表，扩展说明）
**说明**：使用Django自带的分组表，无需额外创建

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | int(11) | 主键，自增 |
| name | varchar(150) | 分组名称，唯一 |

### 2. auth_user（Django自带用户表，扩展说明）
**说明**：使用Django自带的用户表，无需额外创建

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | int(11) | 主键，自增 |
| username | varchar(150) | 用户名，唯一 |
| password | varchar(128) | 加密后的密码 |
| is_superuser | tinyint(1) | 是否超级管理员 |
| is_staff | tinyint(1) | 是否可以登录后台 |
| is_active | tinyint(1) | 账号是否激活 |
| date_joined | datetime(6) | 注册时间 |
| last_login | datetime(6) | 最后登录时间 |

### 3. user_group_membership（用户分组关联表）
**说明**：关联用户和分组，一个用户只能属于一个分组

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int(11) | PRIMARY KEY, AUTO_INCREMENT | 主键 |
| user_id | int(11) | FOREIGN KEY (auth_user.id), UNIQUE | 用户ID，唯一 |
| group_id | int(11) | FOREIGN KEY (auth_group.id) | 分组ID |
| created_at | datetime(6) | DEFAULT CURRENT_TIMESTAMP(6) | 加入时间 |

**索引**：
- UNIQUE INDEX `idx_user_unique` (`user_id`)
- INDEX `idx_group` (`group_id`)

### 4. audio（音频文件表）
**说明**：存储所有上传的音频文件信息

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int(11) | PRIMARY KEY, AUTO_INCREMENT | 主键 |
| filename | varchar(255) | NOT NULL | MP3文件名 |
| file_path | varchar(500) | NOT NULL | 文件在服务器上的存储路径 |
| group_id | int(11) | FOREIGN KEY (auth_group.id), NOT NULL | 所属分组ID |
| upload_time | datetime(6) | DEFAULT CURRENT_TIMESTAMP(6) | 上传时间 |
| file_size | bigint(20) | NULL | 文件大小（字节） |
| duration | int(11) | NULL | 音频时长（秒） |

**索引**：
- INDEX `idx_group_upload_time` (`group_id`, `upload_time` DESC)
- INDEX `idx_upload_time` (`upload_time`)

### 5. user_listened（用户已听记录表）
**说明**：存储用户已听完的音频记录

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int(11) | PRIMARY KEY, AUTO_INCREMENT | 主键 |
| user_id | int(11) | FOREIGN KEY (auth_user.id), NOT NULL | 用户ID |
| audio_id | int(11) | FOREIGN KEY (audio.id) ON DELETE CASCADE, NOT NULL | 音频ID，级联删除 |
| listened_time | datetime(6) | DEFAULT CURRENT_TIMESTAMP(6) | 听完时间 |

**索引**：
- UNIQUE INDEX `idx_user_audio_unique` (`user_id`, `audio_id`)
- INDEX `idx_user_listened_time` (`user_id`, `listened_time` DESC)

## 三、SQL建表语句
```sql
-- 用户分组关联表
CREATE TABLE `user_group_membership` (
  `id` int NOT NULL AUTO_INCREMENT,
  `user_id` int NOT NULL,
  `group_id` int NOT NULL,
  `created_at` datetime(6) NOT NULL DEFAULT CURRENT_TIMESTAMP(6),
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_user_unique` (`user_id`),
  KEY `idx_group` (`group_id`),
  CONSTRAINT `fk_user_group_membership_user` FOREIGN KEY (`user_id`) REFERENCES `auth_user` (`id`) ON DELETE CASCADE,
  CONSTRAINT `fk_user_group_membership_group` FOREIGN KEY (`group_id`) REFERENCES `auth_group` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 音频文件表
CREATE TABLE `audio` (
  `id` int NOT NULL AUTO_INCREMENT,
  `filename` varchar(255) NOT NULL,
  `file_path` varchar(500) NOT NULL,
  `group_id` int NOT NULL,
  `upload_time` datetime(6) NOT NULL DEFAULT CURRENT_TIMESTAMP(6),
  `file_size` bigint DEFAULT NULL,
  `duration` int DEFAULT NULL,
  PRIMARY KEY (`id`),
  KEY `idx_group_upload_time` (`group_id`, `upload_time` DESC),
  KEY `idx_upload_time` (`upload_time`),
  CONSTRAINT `fk_audio_group` FOREIGN KEY (`group_id`) REFERENCES `auth_group` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 用户已听记录表
CREATE TABLE `user_listened` (
  `id` int NOT NULL AUTO_INCREMENT,
  `user_id` int NOT NULL,
  `audio_id` int NOT NULL,
  `listened_time` datetime(6) NOT NULL DEFAULT CURRENT_TIMESTAMP(6),
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_user_audio_unique` (`user_id`, `audio_id`),
  KEY `idx_user_listened_time` (`user_id`, `listened_time` DESC),
  CONSTRAINT `fk_user_listened_user` FOREIGN KEY (`user_id`) REFERENCES `auth_user` (`id`) ON DELETE CASCADE,
  CONSTRAINT `fk_user_listened_audio` FOREIGN KEY (`audio_id`) REFERENCES `audio` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```
