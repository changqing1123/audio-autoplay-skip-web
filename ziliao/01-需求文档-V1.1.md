# MP3分组播放系统 需求文档
**版本**：V1.1  
**更新时间**：2026-05-15  
**技术栈**：Vue 3 + Django 4.x + MySQL 8.x  
**文档状态**：已确认

## 一、项目概述
本项目是一个面向内部用户的MP3音频播放系统，核心功能是按用户分组提供音频播放服务，自动跳过已听完的音频，优先播放最新上传的内容。系统设计追求极简、高效、低服务器压力，仅保留最核心的播放和管理功能。

## 二、核心技术栈
| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Vue 3 + Vue Router | 负责页面路由、播放控制和用户界面 |
| 后端 | Django 4.x + Django REST Framework | 提供API接口和后台管理 |
| 数据库 | MySQL 8.x | 存储用户、分组、音频和已听记录 |
| 后台 | Django自带Admin | 优先使用，后期可自定义美化 |
| 文件存储 | 服务器本地磁盘 | 存储MP3音频文件 |

## 三、核心业务规则
### 3.1 播放顺序黄金规则
**任何时候自动寻找下一首音频，都从列表最顶部（最新上传的）开始，从上往下查找第一首未听完的音频**

### 3.2 已听标记规则
1. 只有当音频**完整播放到100%**时，才会向服务器发送请求，永久标记为"已听完"
2. 一旦标记成功，后台记录永远不会被修改或删除（管理员手动删除除外）
3. 用户手动重复播放已听完的音频，即使再次播放到100%，也不会改变后台记录
4. 手动切歌、暂停、刷新、关闭页面都不会标记为已听完

### 3.3 进度记录规则
1. **数据分离存储**：
   - 服务器端：只存储**已听完的音频ID列表**，不存储任何未完成的播放进度
   - 客户端：本地`localStorage`存储所有未完成音频的播放进度和用户设置
2. **智能进度保存**：
   - 默认**15秒**保存一次进度
   - 提供可配置选项：1秒、5秒、15秒、30秒
   - 关键事件即时保存：暂停、切歌、拖动进度条、关闭页面
3. **进度恢复优先级**：
   - 页面加载时，先从本地读取上次播放的音频ID和进度
   - 如果该音频存在且未被标记为已听完，则从该进度继续播放
   - 如果该音频已被标记为已听完或不存在，则从列表最顶部开始找第一首未听完的音频从0开始播放

### 3.4 分组隔离规则
1. 用户只能看到和播放自己被分配到的分组内的音频
2. 不同分组之间的音频和用户数据完全隔离
3. 音频上传时必须指定所属分组，仅该分组内的用户可见

## 四、前端页面架构
系统采用三页面架构，通过底部导航栏切换：

### 4.1 首页（播放列表页）
- **顶部**：系统标题
- **中间**：音频列表，按上传时间倒序排列（最新的在最上面）
  - 每首音频显示：MP3文件名、上传时间
  - 已听完的音频有明显的视觉区分（如灰色、打勾图标）
  - 点击任意音频立即开始播放
- **底部**：固定的迷你播放控制栏
  - 显示当前播放的歌曲名
  - 播放/暂停按钮、下一首按钮
  - 点击迷你播放栏进入**播放器详情页**

### 4.2 播放器详情页
- 类似网易云音乐的全屏播放器界面
- 显示当前播放的歌曲名和上传时间
- 可拖拽的进度条，显示当前播放时间
- 播放控制按钮：上一首、播放/暂停、下一首
- 返回首页按钮

### 4.3 我的页面
- **已听歌曲列表**：
  - 显示该用户所有已听完的音频，按听完时间倒序排列
  - 点击任意已听音频可以重新播放
- **设置区域**：
  - 进度保存间隔设置（1秒/5秒/15秒/30秒，默认15秒）
  - 首页已听歌曲显示开关（显示/隐藏，默认显示）
  - 退出登录按钮

## 五、功能模块详细说明
### 5.1 用户系统
1. **角色划分**：只有2种角色
   - **超级管理员**：拥有所有权限
     - 创建、删除、修改用户分组
     - 添加、删除、修改用户账号
     - 分配用户到指定分组
     - 上传、删除音频文件
     - 查看和管理所有数据
   - **普通用户**：仅拥有播放权限
     - 登录系统
     - 播放自己所在分组的音频
     - 查看自己的已听列表
     - 修改个人设置
2. **强制登录**：未登录用户无法访问任何页面，自动跳转到登录页
3. **登录方式**：仅支持账号密码登录，不支持第三方登录

### 5.2 播放系统
1. **播放列表**：
   - 首页列表按上传时间倒序排列
   - 支持通过设置隐藏首页的已听歌曲
   - 已听歌曲永远保留在"我的"页面的已听列表中
2. **播放控制**：
   - 基础控制：播放/暂停、上一首、下一首
   - 进度控制：可拖拽的进度条，支持点击跳转
   - 音量控制：使用系统/手机自带音量，网页端不提供音量控制
3. **自动播放逻辑**：
   - 页面加载后自动从上次播放的位置继续播放
   - 一首音频播放完（100%）后，自动标记为已听完
   - 然后从列表最顶部开始，从上往下找第一首未听完的音频播放
   - 一直循环此过程，直到该分组内所有音频都听完为止
4. **上下一首逻辑**：
   - **下一首**：立即停止当前音频，从列表最顶部开始找第一首未听完的音频播放
   - **上一首**：立即停止当前音频，从当前音频的上一个位置开始，往上（更新的方向）找第一首未听完的音频播放
5. **后台播放**：
   - 浏览器标签页在后台运行时，音频继续正常播放
   - 接受浏览器安全限制：必须用户手动点击一次播放按钮后，后台播放才会生效
   - 关闭标签页或浏览器后，播放停止

### 5.3 后台管理系统
1. **用户管理**：添加、删除、修改用户账号，分配用户到分组
2. **分组管理**：创建、删除、修改用户分组
3. **音频管理**：
   - 上传MP3文件到指定分组
   - 上传时自动记录时间戳，用于排序
   - 删除已上传的音频文件
   - 支持按上传时间筛选音频
   - 超过指定时间（如15天）的音频有特殊标记，方便管理员批量清理
4. **自动清理功能**：
   - 支持定时自动清理过期音频（如每天凌晨3点清理15天前上传的音频）
   - 删除音频时，自动级联删除所有用户对应的已听记录
   - 前端自动过滤掉本地存储中已删除音频的进度记录

## 六、非功能需求
1. **性能要求**：支持1000+用户同时在线，服务器无明显压力
2. **可靠性要求**：已听记录永不丢失，播放进度最多丢失15秒（默认设置）
3. **兼容性要求**：支持现代浏览器（Chrome、Safari、Edge）和手机浏览器
4. **安全性要求**：用户密码加密存储，API接口需要认证，防止未授权访问

## 七、边界情况处理
1. **管理员上传新音频**：新音频自动出现在对应分组首页播放列表的最顶部，下一次自动找下一首时会优先检查
2. **音频被删除**：
   - 数据库自动级联删除所有用户对应的已听记录
   - 前端自动过滤掉本地存储中已删除音频的进度记录
   - 如果用户正在播放被删除的音频，会自动停止并跳转到下一首未听完的音频
3. **用户清除浏览器缓存**：
   - 未完成的播放进度和个人设置会丢失
   - 已听完的记录不会丢失，仍然会自动跳过
4. **所有音频都已听完**：自动停止播放，显示"没有更多未听音频"提示
5. **首页隐藏已听歌曲**：
   - 自动播放和上下一首逻辑不受影响，仍然会检查所有音频
   - 只是首页列表不显示已听歌曲，已听歌曲仍然可以在"我的"页面找到
